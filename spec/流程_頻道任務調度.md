# 流程_頻道任務調度

> 本文件描述人類使用者於 Web UI（聊天式工作區）內「檢視頻道階層樹 → 檢視頻道聊天時間軸 → 於頻道發訊（含上傳圖片之純圖訊息）→ 檢視統計儀表板 → 於後台檢視頻道資料」之操作流程, 供 e2e 嚴謹測試與後續報告產製之用. 登入採開發捷徑 token=sys (admin), 進站後預設停在頻道區.

## 觸發

使用者以系統管理者身分登入後進入主畫面. 左1 為近黑導覽 rail, 提供「統計 / 頻道 / 後台」三個區段. 預設停在頻道區: 左2 為階層頻道樹（專案 > 群組 > #頻道）與搜尋框, 點頻道後右側展開聊天工作區（標題列含頻道名 / 麵包屑 / 任務 pills / agent chip, 中央為訊息時間軸, 底部為發訊區）. 使用者點 rail 切換區段; 於頻道區選頻道、檢視時間軸訊息、於發訊區輸入並送出訊息; 於統計區檢視儀表板計數; 於後台區之頻道子頁檢視頻道資料表.

## 重要流程

- **E2E-001**: 登入後預設停在頻道區, 階層頻道樹顯示專案與頻道
  - description: 使用者以 admin (token=sys) 登入後, 主畫面預設停在頻道區, rail 之「頻道」項為選取態. 左2 頻道樹依各頻道之 levels 欄建巢狀階層（專案 > 群組 > #頻道）. 種子含四個頻道, 分屬「數據平台」(ETL 管線 / 報表產製) 與「電商平台改版 > 後端組」(訂單服務 / 金流串接) 兩專案群組. 預期使用者在頻道樹看到此群組階層與其下頻道. 本案例為頻道區初始檢視, 不做任何輸入.
  - flow:
    - 測試資料: 四個頻道種子——`訂單服務`(levels=`電商平台改版.後端組`)、`金流串接`(levels=`電商平台改版.後端組`)、`ETL 管線`(levels=`數據平台`)、`報表產製`(levels=`數據平台`); 由頻道種子提供, 各 case 共用同一 base seed.
    - 操作: 自登入後之主畫面起 (預設頻道區), 不另操作.
    - 驗證: 語意——頁面文字含頻道名 `訂單服務`、`ETL 管線` 與群組名 `數據平台`、`電商平台改版`; 視覺——框左2 頻道階層樹, baseline `test/pics/channeltask/channeltask-{eng,cht}-E2E-001-channels-tree.png`.
    - 雙語: eng / cht 各一輪 (頻道名 / 群組名為資料恆中文, 雙語差異僅在 rail 標籤與樹標題等 i18n chrome).
    - 清理: base seed 為唯讀檢視, 不產生副作用; 由 setup 每 case 重置 DB, 本段不另清理.

- **E2E-002**: 點頻道, 聊天時間軸顯示種子訊息與任務徽章
  - description: 使用者於頻道樹點「訂單服務」頻道, 右側展開該頻道聊天工作區. 種子於該頻道含四則訊息: 人類歡迎語 (text)、agent 上線語 (text)、人類開任務訊息 (task)、agent 任務回應 (taskReply). 預期時間軸由上而下依時序顯示此四則訊息（本人類訊息靠右、agent 訊息靠左）, task 與 taskReply 訊息另以類型徽章標示; 標題列另顯示任務狀態 pills（待處理 / 處理中 / 已完成計數）. 訊息時間戳為固定值以確保畫面穩定.
  - flow:
    - 測試資料: 頻道 `訂單服務` 下四則固定時間戳之種子訊息 (text / text / task / taskReply); 與 base seed 共用.
    - 操作: 自頻道區起 → 點頻道樹中之「訂單服務」頻道列.
    - 驗證: 語意——時間軸文字含 `Welcome to the demo channel.`、`Agent online and ready to take tasks.`、`Please summarize the quarterly report.`、`Summary completed: revenue grew 12% QoQ.`, 並含任務徽章文字（eng: `Task` / `Task reply`; cht: `任務` / `任務回應`）與標題列任務 pills（eng: `Pending` / `Done`; cht: `待處理` / `已完成`）; 視覺——框聊天時間軸區, baseline `test/pics/channeltask/channeltask-{eng,cht}-E2E-002-chat-view.png`.
    - 雙語: eng / cht 各一輪.
    - 清理: 唯讀檢視無副作用; 由 setup 每 case 重置 DB, 本段不另清理.

- **E2E-003**: 於發訊區輸入訊息並送出, 新訊息出現於時間軸
  - description: 使用者於頻道「訂單服務」之底部發訊區輸入一則文字訊息並以 Enter 送出. 預期送出成功後輸入內容出現於時間軸（本人類訊息靠右）, 且發訊區輸入框清空. 本案例為真實輸入與送出之操作鏈, 重點驗證「輸入 → 送出 → 出現 + 清空」之語意鏈. 因新訊息帶即時時間戳, 視覺 baseline 改對「發訊區（輸入框 + 工具列 + 送出鈕）」於送出前（內容填妥）框紅框截圖, 避開動態時間戳區.
  - flow:
    - 測試資料: 頻道 `訂單服務`; 與 base seed 共用. 本案例會於該頻道新增一則人類文字訊息 (副作用).
    - 操作: 自頻道區起 → 點頻道樹中之「訂單服務」頻道列 → 於底部發訊區輸入框輸入一段文字 → 按 Enter 送出.
    - 驗證: 語意——送出後時間軸文字含剛輸入之訊息內容, 且發訊區輸入框值為空; 視覺——框底部發訊區（輸入框 + 工具列 + 送出鈕, 內容填妥送出前, 避開含即時時間戳之時間軸）, baseline `test/pics/channeltask/channeltask-{eng,cht}-E2E-003-chat-send.png`.
    - 雙語: eng / cht 各一輪 (輸入內容含語系後綴, 兩語系 baseline 之發訊區內容與按鈕文字各異).
    - 清理: 移除本案例新增之訊息, 保留 base seed; 由 setup 每 case 重置 DB 達成 (刪 db 重建種子).

- **E2E-004**: 切統計區, 儀表板顯示概況計數
  - description: 使用者點 rail 之「統計」項切到統計區. 統計儀表板上半 OVERVIEW 區顯示三張卡——頻道數 / 訊息數 / 任務總數; 下半 TASK BREAKDOWN 區顯示任務各狀態計數（待處理 / 處理中 / 已完成 / 失敗）. 計數來源為後端統計, 對 pristine 種子為固定值（頻道 4、訊息 4、任務 3）. 預期 OVERVIEW 區呈現三卡標題與其計數值.
  - flow:
    - 測試資料: base seed（四頻道 / 四訊息 / 三任務）; 與 base seed 共用.
    - 操作: 自頻道區起 → 點 rail 之「Stats」/「統計」項切到統計區.
    - 驗證: 語意——OVERVIEW 區文字含三卡標題（eng: `Channels` / `Messages` / `Tasks`; cht: `頻道數` / `訊息數` / `任務總數`）及計數值（頻道 4、任務 3）; 視覺——框 OVERVIEW 卡片區, baseline `test/pics/channeltask/channeltask-{eng,cht}-E2E-004-stats.png`.
    - 雙語: eng / cht 各一輪.
    - 清理: 唯讀檢視無副作用; 由 setup 每 case 重置 DB, 本段不另清理.

- **E2E-005**: 切後台區, 頻道子頁資料表顯示階層欄與頻道列
  - description: 使用者點 rail 之「後台」項切到後台區, 後台左2 子選單預設選取「頻道」子頁, 右側以資料表（ag-grid）列出全部頻道. 資料表含「頻道名 / 階層 / 頻道描述 / agent / 擁有者」等欄. 預期資料表顯示階層 (Levels) 欄表頭與四個頻道列, 各頻道之階層欄呈現其 levels 值. 本案例為後台資料檢視.
  - flow:
    - 測試資料: 四個頻道種子（含 levels 欄值）; 與 base seed 共用.
    - 操作: 自頻道區起 → 點 rail 之「Admin」/「後台」項 → 後台預設停在「頻道」子頁.
    - 驗證: 語意——資料表文字含階層欄表頭（eng: `Levels`; cht: `階層`）、頻道名 `訂單服務` 與其階層值 `電商平台改版.後端組`; 視覺——框後台頻道資料表, baseline `test/pics/channeltask/channeltask-{eng,cht}-E2E-005-admin-channels.png`.
    - 雙語: eng / cht 各一輪.
    - 清理: 唯讀檢視無副作用; 由 setup 每 case 重置 DB, 本段不另清理.

- **E2E-006**: 於發訊區上傳圖片並送出純圖訊息, 圖片出現於時間軸
  - description: 使用者於頻道「訂單服務」之發訊區點附圖鈕, 選取一張本地圖片, 發訊區出現該圖之預覽縮圖. 使用者不輸入任何文字, 直接點送出, 送出一則「純圖訊息」(內容空、僅帶附件). 預期送出成功後時間軸出現一則新訊息且含該圖片附件, 發訊區輸入框維持為空. 本案例驗證「貼多圖」之圖台互動鏈與「純圖訊息」(內容空時仍可憑附件送出) 之行為. 因新訊息之附件帶即時產生之檔案識別與時間戳, 視覺 baseline 改對「發訊區預覽縮圖區」於送出前框紅框截圖, 避開動態值區.
  - flow:
    - 測試資料: 頻道 `訂單服務`; 與 base seed 共用. 本案例會於該頻道新增一則含圖片附件之人類訊息 (副作用), 並於圖台新增一筆檔案.
    - 操作: 自頻道區起 → 點頻道樹中之「訂單服務」頻道列 → 於發訊區點附圖鈕並選取一張本地圖片 → 等預覽縮圖出現 → 不輸入文字 → 點「Send」/「送出」按鈕.
    - 驗證: 語意——送出後時間軸含一則帶圖片附件之訊息（附件圖以頻道檔案串流呈現）, 且發訊區輸入框值為空; 視覺——框發訊區預覽縮圖區（內容備妥送出前, 避開含即時檔案識別 / 時間戳之時間軸）, baseline `test/pics/channeltask/channeltask-{eng,cht}-E2E-006-chat-upload-image.png`.
    - 雙語: eng / cht 各一輪.
    - 清理: 移除本案例新增之訊息與圖台檔案, 保留 base seed; 由 setup 每 case 重置 DB（刪 db 重建種子、清圖台檔案表與實體目錄）達成.

- **E2E-007**: 送出挾帶 HTML 之訊息, 內容經消毒後安全渲染不執行 JS
  - description: 使用者於頻道「訂單服務」之發訊區輸入一段挾帶惡意 HTML（如帶 `onerror` 之圖片標籤）之訊息並送出. 預期時間軸渲染該訊息時, 危險之 inline event handler 被消毒剝除、不執行任何 JS, 但純文字內容仍保留. 本案例為儲存型 XSS 之回歸守衛（訊息內容允許 markdown、以富文字方式渲染, 渲染前須先消毒）. 因渲染後訊息帶即時時間戳, 且「已消毒」與「未消毒」之純文字視覺無從以像素區辨, 本案例為語意-only（不另存視覺 baseline; 發訊區與時間軸之視覺已由 E2E-002 / E2E-003 覆蓋）.
  - flow:
    - 測試資料: 頻道「訂單服務」; 與 base seed 共用. 本案例會於該頻道新增一則含 HTML 之人類訊息（副作用）.
    - 操作: 自頻道區起 → 點頻道樹中之「訂單服務」頻道列 → 於發訊區輸入挾帶惡意 HTML 之訊息 → 按 Enter 送出.
    - 驗證: 語意——(1) 惡意 handler 未執行（無 JS 副作用）; (2) 時間軸渲染後之 HTML 不含 `onerror` 屬性; (3) 訊息之純文字內容仍保留; 視覺——語意-only, 不另存 baseline（理由見 description）.
    - 雙語: eng / cht 各一輪（消毒與語系無關, 兩語系下皆須成立）.
    - 清理: 移除本案例新增之訊息, 保留 base seed; 由 setup 每 case 重置 DB 達成.

- **E2E-008**: 於頻道聊天頁開啟本頻道成員管理, 完成成員新增 / 更新 / 刪除
  - description: 使用者於頻道「訂單服務」之聊天頁標題列點「成員」入口, 開啟本頻道成員管理視窗. 因視窗綁定當前頻道, 不再顯示頻道選擇器（免再選頻道即可管理該頻道成員）. 視窗載入該頻道成員 agent-demo, 成員類型欄以在地化文字顯示（agent → `Agent`; human → `Human` / `人類`）. 使用者切換編輯模式, 完成一輪完整成員 CRUD: 新增一列（預設類型 human）並儲存（Create）; 於資料表就地編輯該成員之角色欄並儲存（Update）; 勾選該成員、按刪除並確認（Delete）. 本案例為承接式 journey（開啟 → 新增 → 更新 → 刪除之狀態逐步推進, 後段承接前段真實副作用, 屬單一 case 多階段）. 每階段以「頻道成員實際內容」觀察後端副作用（成員增 / 改 / 刪）. 視覺 baseline 對檢視模式之成員視窗（僅 agent-demo 一列, 種子時間戳固定 → 穩定）截圖; CRUD 各階段帶動態成員識別碼不另存視覺.
  - flow:
    - 測試資料: 頻道「訂單服務」與其主責成員 agent-demo; 與 base seed 共用. 本案例新增後又刪除自身測試成員, 最終回歸 pristine.
    - 操作: 自頻道區起 → 點頻道樹中之「訂單服務」頻道列 → 點標題列「成員」入口 → 切「編輯模式」開關 → 點「新增」→ 點「儲存」並於成功視窗按確定（Create）→ 於新列角色欄就地輸入新值 → 點「儲存」並確定（Update）→ 勾選該列 → 點「刪除」→ 於確認視窗按「Yes」/「確定」→ 於成功視窗按確定（Delete）.
    - 驗證: 語意——(1) 視窗載入本頻道成員 agent-demo; (2) 新增後該頻道成員數為 2（新成員為 human, 其類型欄在地化顯示 `Human` / `人類`, agent-demo 顯示 `Agent`）; (3) 更新後該成員角色欄之值變為輸入之新值; (4) 刪除後該頻道成員數回 1, 僅剩 agent-demo; (5) 綁定當前頻道時不顯示頻道選擇器; 視覺——框成員視窗（檢視模式, agent-demo 一列）, baseline `test/pics/channeltask/channeltask-{eng,cht}-E2E-008-channel-members-crud.png`.
    - 雙語: eng / cht 各一輪.
    - 清理: 本案例最終刪除自身新增之成員, DB 回歸 pristine（僅 agent-demo）; setup 每 case 重置 DB 亦保底還原.

- **E2E-009**: 後台管理之任務子頁, 資料表顯示頻道任務清單與狀態
  - description: 使用者切至後台管理區, 點「任務」子頁. 任務子頁以頻道選擇器預設頻道（訂單服務）載入其任務清單, 資料表顯示各任務之標題、狀態、認領者與時間. 狀態欄以在地化文字並依狀態上色（待認領 / 執行中 / 已完成 / 失敗）. 工具列另有「顯示封存」切換（預設關, 開啟時資料來源改為封存冷表之唯讀檢視）, 本案例維持預設關、驗證熱表資料表呈現. 種子任務之時間欄為固定值 → 視覺穩定.
  - flow:
    - 測試資料: 頻道「訂單服務」之三筆種子任務（一 done、一 pending、一 error, 時間欄固定）; 與 base seed 共用. 唯讀檢視, 無副作用.
    - 操作: 自任一區起 → 點左側「後台」→ 點子選單「任務」.
    - 驗證: 語意——資料表含三筆種子任務標題（`Summarize quarterly report`、`Draft the release notes`、`Fetch external metrics`）, 且狀態欄含三態之在地化值（eng: `Done` / `Pending` / `Error`; cht: `已完成` / `待認領` / `失敗`）; 視覺——框後台任務資料表, baseline `test/pics/channeltask/channeltask-{eng,cht}-E2E-009-admin-tasks.png`.
    - 雙語: eng / cht 各一輪.
    - 清理: 唯讀檢視無副作用; setup 每 case 重置 DB 仍還原 pristine 種子.

- **E2E-010**: 後台管理之成員子頁, 資料表顯示頻道成員（含在地化類型）與頻道選擇器
  - description: 使用者切至後台管理區, 點「成員」子頁. 成員子頁以頻道選擇器預設頻道（訂單服務）載入其成員清單, 資料表顯示成員識別碼、類型（在地化）、角色等. 因後台入口未綁定頻道, 頻道選擇器可見（與 E2E-008 頻道聊天頁入口綁定頻道 → 隱藏選擇器互補: 同一元件兩入口之差異）. 種子成員之時間欄為固定值 → 視覺穩定.
  - flow:
    - 測試資料: 頻道「訂單服務」之主責成員 agent-demo（時間欄固定）; 與 base seed 共用. 唯讀檢視, 無副作用.
    - 操作: 自任一區起 → 點左側「後台」→ 點子選單「成員」.
    - 驗證: 語意——(1) 資料表含成員 agent-demo; (2) 成員類型欄在地化顯示 `Agent`; (3) 後台入口顯示頻道選擇器（與 E2E-008 綁定隱藏互補）; 視覺——框後台成員資料表, baseline `test/pics/channeltask/channeltask-{eng,cht}-E2E-010-admin-members.png`.
    - 雙語: eng / cht 各一輪.
